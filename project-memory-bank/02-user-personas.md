# User Personas

## Primary — "The Founder" (build for this person first)
| Attribute | Detail |
|-----------|--------|
| Role | Lead Engineer, aspiring Staff / Principal |
| Context | Works on real, ambiguous, high-stakes technical problems daily |
| Interests | AI, knowledge systems, semantic control planes, engineering intelligence platforms |
| Motivation | Wants to *systematically* level up judgment, not wait for it to accrete by accident |
| Frustration | No structured daily practice exists; reflection is ad hoc and forgotten |
| Environment | Often thinks while offline / focused / away from network — **offline-first is a real need, not a nicety** |

### Jobs To Be Done (founder)
- "When I face a messy problem, give me a fast way to **frame** it before I jump to solutions."
- "When I make a consequential decision, help me **record my reasoning + confidence** so I can review whether I was right."
- "When I sense a system is fragile, help me **map it** and find bottlenecks/failure modes."
- "Each week, help me **see patterns** in how I think and where I'm weak."

### Daily Usage Scenario
Founder opens EJOS in the morning or before a hard meeting. App shows: *"What problem are you thinking about today?"* They spend 5–15 minutes framing it (problem tree, assumptions, stakeholders). The output is saved to their Cognitive Repository. Over weeks, the journal becomes a mirror of their evolving judgment. No network required for any of this.

## Secondary Personas (later — do NOT build for these yet)
- Senior engineers preparing for Staff promotion.
- Eng leaders / managers coaching their teams.
- These inform direction but must not pull scope away from founder daily usage.
